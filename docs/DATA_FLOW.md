# 상품 데이터 흐름 (세트 데이터 원칙)

## 원칙

- **AI 추출된 정보는 한 세트**로 다 같이 이동한다.
- 이동·저장 시 **데이터가 정확히 넘어가야** 하며, **항상 세트 데이터**로 다룬다.
- **등록대기 리스트**에 올라온 항목 = AI 추출 세트가 저장된 상태.
- 등록대기 목록에 **사진이 더해지면** → **완전한 데이터**가 된다 (검수 후 등록 가능).

## 흐름 요약

1. **상품 등록 (AI 추출)**  
   - 여행사 텍스트 붙여넣기 → AI 추출 → 상품정보·가격·일정 등 **한 세트**로 파싱.
2. **저장**  
   - 파싱된 세트 전체를 **한 번에** DB로 저장(Upsert by originCode).  
   - 누락 없이, 같은 상품이면 **세트 단위로 갱신** (기존 가격·일정 삭제 후 새 세트로 교체).
3. **등록대기 리스트**  
   - 저장된 **세트 데이터**가 등록대기 목록에 노출.  
   - 이 단계 = “모인 정보”가 DB에 반영된 상태.
4. **사진 추가**  
   - process-images 등으로 메인·일정 이미지가 채워지면 **완전한 데이터**.
5. **검수 후 등록**  
   - 완전한 데이터를 검수한 뒤 등록 → 상품 목록에 노출.

## 구현 시 유의사항

- 저장·업데이트 시 **parsed 세트 전체**를 기준으로 Product + ProductPrice + Itinerary 를 한 트랜잭션에서 갱신.
- 부분 수정이 아니라 **세트 단위 replace** 로 해서, 이동 시 데이터가 정확히 넘어가도록 유지.
