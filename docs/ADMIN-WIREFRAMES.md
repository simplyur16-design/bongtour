# 봉투어 관리자 시스템 — 텍스트 와이어프레임

**전제**: B2C 상담유도형. 직접예약·결제 없음. 고객은 상품 조회 후 상담예약/문의. 상담 채널은 카카오 오픈채팅 중심. 관리자는 **상품 등록/검수/노출/상담 접수** 중심.

**메뉴 6개**: 대시보드, 상품 등록, 등록대기, 상품목록, 상담·예약, 스케줄러·보안.

---

## 1. 전체 와이어프레임 원칙

### 레이아웃

- **좌측 사이드바**: 고정 너비(접기 가능). 메뉴 6개만 표시. 아이콘 + 라벨. 활성 메뉴는 배경/색 강조. 상단에 "봉투어 관리" 로고/텍스트.
- **메인 영역**: 사이드바 오른쪽 전체. 배경은 밝은 회색 또는 흰색. 패딩 일관 유지.
- **상단 헤더**: 각 페이지 최상단에 페이지 제목(H1) + 서브 문구 한 줄. 테두리 또는 구분선으로 본문과 분리 가능.

### 페이지 제목 영역

- **H1**: 페이지 목적을 한 문장으로. 예: "대시보드", "상품 등록", "등록대기".
- **서브 문구**: 보조 설명 한 줄. 예: "오늘 현황과 빠른 작업", "저장 후 등록대기에서 이미지·검수를 진행하세요."
- 제목 영역 아래 여백 후 본문 시작.

### KPI 카드 규칙

- **위치**: 페이지 상단, 제목/서브 문구 바로 아래.
- **형태**: 가로 배치 카드 2~4개. 각 카드: 라벨(작은 글씨) + 숫자(강조) 또는 짧은 문구. 클릭 시 해당 메뉴/필터로 이동 가능하면 링크 스타일.
- **내용**: 건수·상태 요약만. 상세 테이블은 카드에 넣지 않음.
- **빈 상태**: 데이터 없을 때 "0건" 등으로 표시. 문구는 빈 상태 섹션에서 통일.

### 표/리스트 규칙

- **테이블**: thead(컬럼명) + tbody(행). 정렬 가능 컬럼은 헤더에 표시(선택). 행 호버 시 배경 변경. 행 클릭 시 상세/모달 가능하면 명시.
- **리스트**: 카드형 또는 단순 목록. 각 항목에 최소 식별자(이름/코드/날짜) + 상태 배지 + 액션 버튼.
- **페이징**: 목록 하단. "이전 / 1 2 3 / 다음" 또는 무한 스크롤 명시.
- **빈 목록**: 테이블/리스트가 비었을 때 가운데 정렬 문구. "등록대기 상품이 없습니다." 등.

### 상태 배지 규칙

- **위치**: 목록 행 내 또는 상세 헤더 옆.
- **형태**: 작은 칩/뱃지. 텍스트만(접수완료, 상담중, 노출, 비노출 등). 색은 상태 톤만 구분(녹색/회색/주황/빨강 등 톤만 문서에 명시).
- **일관성**: 같은 상태값은 프로젝트 전역에서 동일 라벨·동일 톤 사용.

### 빠른 액션 버튼 규칙

- **위치**: 제목 아래 또는 KPI 카드 아래. "빠른 액션" 또는 "주요 작업" 섹션으로 그룹.
- **형태**: 1차 액션은 채움 버튼(진한 배경). 2차는 테두리만. 작은 액션은 텍스트 링크.
- **순서**: 자주 쓰는 것부터 왼쪽. 예: [상품 등록] [등록대기 보기] [상품 목록 보기].

### 빈 상태 / 에러 상태 / 성공 상태 메시지

- **빈 상태**: "~이(가) 없습니다." + 다음 행동 안내 한 줄. 예: "상품 등록에서 새 상품을 저장해 주세요."
- **에러 상태**: "~을(를) 불러오지 못했습니다." 또는 "저장 실패: {원인}". 빨간 톤 또는 경고 영역.
- **성공 상태**: "저장되었습니다.", "승인되었습니다." 등 짧게. 녹색 톤 또는 성공 영역. 토스트 또는 인라인 블록 중 하나로 통일.

### 공통 금지 사항

- 결제/PG/예약금/잔금 입력 UI 넣지 않음.
- 고객용 예약 엔진처럼 "결제하기"/"입금 확인" 버튼 넣지 않음.
- 관리자 메인 플로우와 무관한 대형 차트/대시는 대시보드 외 페이지에 넣지 않음.

### [대시보드] `/admin`

**1) 페이지 목적**  
관리자 첫 화면. 오늘 숫자 확인 후 필요한 메뉴로 빠르게 이동.

**2) 사용자 역할**  
운영 담당자. 로그인 후 "뭘 먼저 할까"를 확인하고 등록대기·상품 등록·상담 접수 등으로 점프.

**3) 상단 헤더 문구**  
"봉투어 관리자"

**4) 서브 문구**  
"오늘 현황과 빠른 작업"

**5) 상단 KPI/요약 카드**  
- 등록대기 N건 (클릭 시 /admin/pending)  
- 상품 목록 N건 (클릭 시 /admin/products)  
- 상담 접수 N건 (클릭 시 /admin/bookings)  
- 오늘 수집: 성공 K / 실패 L (또는 "아래 차트 참고")

**6) 주요 섹션 순서**  
1. KPI 카드  
2. 빠른 액션  
3. 빈 상태 안내(등록대기·상품 모두 0일 때만)  
4. 오늘 수집 현황·봇 상태·가격 동기화 1회·로그

**7) 각 섹션 요소**  
- **빠른 액션**: [상품 등록] [등록대기 보기] [상품 목록 보기] [상담·예약 보기] [브랜드 관리]  
- **오늘 수집**: 봇 상태 인디케이터, 오늘 수집 성공/실패 차트, [지금 1회 실행] 버튼, 실시간 로그 터미널

**8) 주요 버튼**  
상품 등록, 등록대기 보기, 상품 목록 보기, 상담·예약 보기, 브랜드 관리, (로그 영역 내) 지금 1회 실행

**9) 표/리스트**  
없음. 카드와 버튼만.

**10) 상태 배지**  
없음.

**11) 빈 상태 문구**  
"등록대기 0건, 상품 0건. 상품 등록에서 첫 상품을 추가해 보세요."

**12) 에러 상태 문구**  
"데이터를 불러오지 못했습니다. 새로고침해 주세요."

**13) 성공 상태 문구**  
(이 페이지에서 저장 없음. 가격 동기화 1회 시 "요청을 보냈습니다." 등 토스트만.)

**14) 다른 페이지로 이동**  
위 빠른 액션으로 /admin/register, /admin/pending, /admin/products, /admin/bookings, /admin/brands. 로그 내 run once는 API 호출만.

**15) 넣지 말 것**  
텍스트 붙여넣기·상품 저장 폼, 시세 2주 테이블, 상담 포인트 상세, 등록 상품 전체 테이블, 예약 상세 편집 폼.

**16) ASCII 와이어프레임**

### [상품 등록] `/admin/register` (상세)

**1) 페이지 목적**  
상품 한 건을 URL 또는 텍스트/HTML로 입력 → AI 분석 → 저장 → 등록대기로 보내는 **등록 전용** 화면.

**2) 사용자 역할**  
운영 담당자. 여행사 상세 텍스트를 복사해 붙여넣고, 여행사 선택 후 한 번에 분석·저장해 등록대기로 넘기려 할 때 사용.

**3) 상단 헤더 문구**  
"상품 등록"

**4) 서브 문구**  
"URL 또는 상세 텍스트를 붙여넣어 AI가 분석한 뒤 저장합니다. 저장 후 등록대기에서 이미지·검수·분류 확정을 진행하세요."

**5) 상단 KPI/요약 카드**  
없음. 입력·버튼·결과만.

**6) 주요 섹션 순서**  
1. 여행사 선택  
2. (향후) URL 입력 영역 — 보류 시 placeholder 또는 "URL 입력 (준비 중)" 영역  
3. 텍스트/HTML 붙여넣기 입력창  
4. [AI 분석 및 상품 등록] 버튼  
5. 상태/진행 메시지  
6. 저장 성공 시: 메시지 + [등록대기로 이동] + [상품 상세 보기]  
7. 1차 자동 추출 분류 표시 영역(국가·도시·여행타입·쇼핑·가이드경비·선택관광 등)  
8. 하단 링크: ← 대시보드, 등록대기 보기

**7) 각 섹션 요소**  
- **여행사 선택**: 드롭다운(브랜드 목록). 설명: "붙여넣을 텍스트의 출처 여행사를 선택하세요."  
- **URL 입력(보류)**: 입력 필드 1개 + "URL에서 가져오기 (준비 중)" 또는 회색 placeholder.  
- **입력창**: 한 개 textarea. placeholder: "여행사 상세페이지에서 복사한 텍스트 또는 HTML을 붙여넣으세요. AI가 상품코드·가격·일정을 추출합니다."  
- **버튼**: "AI 분석 및 상품 등록" (로딩 시 "분석 및 저장 중…")  
- **상태**: "상태" 라벨 + 진행 문구(제미나이 분석 중… / 픽셀스 이미지 검색 중… / DB 저장 중…) 또는 "저장되었습니다. 등록대기에서 이미지 수급 후 승인해 주세요."  
- **성공 시**: [등록대기로 이동] (1차 버튼) | [상품 상세 보기] (2차 링크)  
- **1차 분류 표시**: 자동 추출된 값만 표시. 편집은 등록대기에서.  
  - 국가, 도시, 여행타입, 쇼핑 횟수, 가이드경비 포함 여부, 선택관광 요약 등.  
  - "1차 자동 분류 (등록대기에서 확정)" 라벨.

**8) 주요 버튼**  
AI 분석 및 상품 등록, 등록대기로 이동, 상품 상세 보기, ← 대시보드, 등록대기 보기

**9) 표/리스트**  
없음. 1차 분류는 키-값 블록 또는 작은 테이블로만 표시.

**10) 상태 배지**  
없음. 진행 중/완료/에러는 텍스트로만.

**11) 빈 상태 문구**  
"텍스트를 입력하세요." (버튼 클릭 시 검증)

**12) 에러 상태 문구**  
"상품코드(originCode)를 추출할 수 없습니다." / "등록 실패" / "파싱 또는 등록 실패"

**13) 성공 상태 문구**  
"저장되었습니다. 등록대기에서 이미지 수급 후 승인해 주세요."

**14) 다른 페이지로 이동**  
저장 성공 후: 등록대기(/admin/pending), 상품 상세(/admin/products/[id]). 하단: 대시보드(/admin), 등록대기 보기(/admin/pending).

**15) 넣지 말 것**  
등록대기/상품목록/대기열 테이블, 시세 관제, 상담 포인트 목록, 예약 현황, 스케줄러 설정, 결제/PG UI.

### [등록대기] `/admin/pending` (상세)

**1) 페이지 목적**  
저장된 상품의 이미지 수급·2차 분류 확정·검수 후 승인해 상품목록으로 승격.

**2) 사용자 역할**  
운영 담당자. 등록대기 목록에서 이미지 채우기, 분류 확정, 승인/반려/보류.

**3) 상단 헤더 문구**  
"등록대기"

**4) 서브 문구**  
"저장된 상품에 이미지를 채우고, 분류를 확정한 뒤 승인하면 상품 목록(고객 노출)으로 이동합니다."

**5) 상단 KPI/요약 카드**  
등록대기 N건

**6) 주요 섹션 순서**  
1. 요약 카드(건수)  
2. 대기 상품 리스트(테이블 또는 카드)  
3. 행별: 이미지 필요 여부, 이미지 소스 선택, 2차 분류 확정, 승인/반려/보류

**7) 각 섹션 요소**

- 고객 메인 노출 여부, 상담 유도 운영용 상태.  
- 대량 액션: 선택 N건 가격 동기화 등 판단 후 반영.

## 8. 상품 상태 / 상담 상태 설계

### 상품 상태

| 상태 | 의미 | 보이는 페이지 | 다음 액션 | 배지 톤 |
|------|------|----------------|-----------|---------|
| pending | 저장됨, 등록 전 | 등록대기 | 이미지 수급, 분류 확정, 승인 | 주황 |
| pending_image | 이미지 수급 대기(선택) | 등록대기 | 이미지 채우기, 승인 | 주황 |
| pending_review | 검수 대기(선택) | 등록대기 | 승인/반려 | 노랑 |
| registered | 승인 완료, 상품목록에 있음 | 상품목록 | 가격 동기화, 노출 on/off, 상세 수정 | 녹색 |
| hidden | 목록에 있으나 고객 비노출 | 상품목록 | 노출 on | 회색 |
| error | 수집/파싱 오류 | 상품목록(필터) | 재시도, 수정 | 빨강 |

### 상담 상태

| 상태 | 의미 | 다음 액션 | 배지 톤 |
|------|------|-----------|---------|
| 접수완료 | 접수됨, 미처리 | → 상담중, 취소 | 회색/기본 |
| 상담중 | 상담 진행 중 | → 예약확정, 취소 | 파랑 |
| 예약확정 | 담당자 확정·상담 완료(결제 아님) | → 취소(필요 시) | 녹색 |
| 취소 | 취소됨 | (종료) | 회색 |

## 9. 페이지 간 이동 규칙

| 상황 | 이동 대상 |
|------|-----------|
| 대시보드 → 상품 등록 | /admin/register |
| 대시보드 → 등록대기/상품목록/상담·예약 | /admin/pending, /admin/products, /admin/bookings |
| 상품 등록 저장 성공 | 현재 페이지 유지. [등록대기로 이동] → /admin/pending, [상품 상세 보기] → /admin/products/[id] |
| 등록대기 승인 | 해당 행 제거 또는 "상품 목록에서 보기" 안내. 이동: /admin/products |
| 상품목록 상세 수정 | /admin/products/[id] |
| 상담·예약에서 관심 상품으로 | /admin/products/[id] (선택) |
| 스케줄러 run once 후 결과 확인 | 대시보드 "오늘 수집" 또는 스케줄러 페이지 내 로그 |
| /admin/control 접속 | /admin 으로 리다이렉트 |

## 10. 최종 구현 우선순위

1. **대시보드 고정** — KPI 카드·빠른 액션·오늘 수집 블록 유지.  
2. **상품 등록 구조 정리** — 여행사 선택, 1차 분류 표시 영역, URL placeholder, 저장 후 등록대기/상품 상세만 노출.  
3. **등록대기 설계 반영** — 이미지 소스 선택(Pexels/Gemini/업로드), 2차 분류 확정, 승인/반려/보류.  
4. **상품목록 운영 요소** — 검색/필터, 노출 on/off, 가격 동기화, 이미지 재처리, (선택) 대량 액션.  
5. **상담·예약 톤 정리** — 상담 접수형 문구·상태만. 결제/PG 없음 유지. 오픈채팅/메모(개념) 반영.  
6. **스케줄러·보안** — Cron·run once·비상정지·옵션·로그 유지, 상품/회원 기능 추가 금지.
