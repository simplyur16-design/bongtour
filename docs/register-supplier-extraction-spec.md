# 관리자 등록: 공급사별 상세 추출 대상 · 입력 규약 · 우선순위

## 문서 성격 (고정)

- 본 문서는 **현재 레포에 존재하는 경로·동작을 설명·규약화한 것**이다. “앞으로도 공용/공통 파이프를 표준으로 유지한다”는 뜻이 **아니다**.
- **구현 방향(고정):** 공급사별로 **전용 파이프만** 목표로 한다.  
  `modetour` → 모두투어 전용 · `verygoodtour` → 참좋은여행 전용 · `ybtour`(레거시 키 `yellowballoon`) → 노랑풍선 전용 · `hanatour` → 하나투어 전용.
- 아래 **공용·레거시 파일**은 “새 기준”이 아니라, **레포에 남아 있는 잔재**이며 표에서 **보류 / 축소 대상 / 삭제 후보** 중 **하나**로만 분류한다.

## 근거 텍스트 (샘플)

- 붙여넣기 형태의 구체 문장·표 헤더는 레포 회귀 픽스처 `scripts/verify-detail-body-parser-regression.ts`의 `samples[]`(id `M1`–`M3`, `C1`–`C7`, `H1`–`H3`, `Y1`–`Y3`)와, 관리자 UI 프레임 `lib/admin-register-supplier-input-frames.ts`의 placeholder를 **근거 문자열**로 삼는다.
- 운영 세부 규약·placeholder 정렬: `docs/admin-register-supplier-precise-spec.md`.

## 본문 섹션 분리·추출 (공급사별 SSOT)

회귀 픽스처·UI 프레임 근거는 본 문서와 `admin-register-supplier-precise-spec.md`에 두되, **앵커·정규화·섹션 슬라이스**의 유지보수 1급 기준은 각 `docs/body-parser-hanatour-ssot.md`, `docs/body-parser-modetour-ssot.md`, `docs/body-parser-verygoodtour-ssot.md`, `docs/body-parser-ybtour-ssot.md`이며, 대응 코드는 해당 파일 상단 `@see`와 문서 내 코드 목록이 양방향으로 맞춘다.

## 공급사 공통 입력 우선순위 규칙

관리자 **5개 입력칸**(본문·항공·호텔·옵션·쇼핑)과 LLM·파서의 역할 분담. **전용 파이프별로** 동일 원칙을 적용한다.

| 구분 | 담당 | SSOT |
|------|------|------|
| 본문 입력칸 | LLM + 결정적 섹션 분리(일정·포함불포함 등) | 서술형 일정·관광·식사·이동·포함/불포함·주의·미팅 등 **텍스트·구조화된 schedule/inc-exc** |
| 항공 정형칸 | 결정적 파서 + (필요 시) 본문 항공 구간 보조 | **정형칸에 값이 있으면 정형칸이 항공 슬롯의 최종 우선** |
| 호텔 정형칸 | 표/리스트 파서 | **정형칸이 있으면 호텔 행·후보의 최종 우선**; 본문 호텔 문구는 비고·보조 |
| 옵션 정형칸 | 표 파서 + LLM 보조 JSON | **옵션관광 최종 SSOT = 정형 옵션칸** (행 JSON + 상한 정책). 본문/LLM은 보조·검수만 |
| 쇼핑 정형칸 | 표 파서 + LLM 보조 | **쇼핑 행 리스트 SSOT = 정형 쇼핑칸**; `shoppingVisitCount`는 요약·별도 필드로 유지 |

### 충돌 시 원칙

1. **항공 — 본문 `flight_section` vs 항공칸**  
   정형 항공칸에 의미 있는 값이 있으면 **병합하되 확정·미리보기·저장은 정형칸에 유리**. 정형칸이 비었을 때만 본문 구조화·LLM 항공을 신뢰.
2. **호텔 — 후보 목록 vs 설명형**  
   **표/행 우선**. 설명형만 있으면 `hotel_info` 텍스트·검수 이슈로 보존.
3. **쇼핑 — 횟수 vs 리스트**  
   **행 리스트는 잘리지 않게 보존**. 횟수 불일치 시 `shoppingVisitCount`만 정합·검수. 도시별·회차별 행 **중복 제거 금지**.
4. **옵션**  
   **최종 SSOT = 정형 옵션칸**. 도시별 반복 **중복 제거 금지**. 최대 **30행**은 `OPTIONAL_TOUR_UI_MAX_ROWS` 등과 동기.
5. **본문 LLM 금지**  
   정형칸이 채운 축의 **최종 값을 덮어쓰기·대체하지 않음**.

## 관리자 입력값 규약 (5축)

UI 축은 공통이나, **넣는 형식은 공급사별**로 `lib/admin-register-supplier-input-frames.ts`·`docs/admin-register-supplier-precise-spec.md`와 일치시킨다.

| 축 | 들어가야 할 것 |
|----|----------------|
| **본문** | 해당 브랜드 **서술형 SSOT** (일정·관광·식사·이동·포함불포함·주의·미팅·참좋은 여정 요약 등). 항공/호텔/옵션/쇼핑 전문 중복은 선택; 정형칸이 있으면 정형 우선. |
| **항공** | 출국·입국 또는 출발·도착 **블록 원문**; 항공사·편명·도시·일시. |
| **호텔** | 일자·도시·호텔명 **표/리스트** 또는 후보·미정 설명. |
| **옵션** | 옵션 **표·블록 원문** (성인/아동·시간·대기·동행 등). |
| **쇼핑** | 쇼핑 **표/리스트**; 횟수는 본문/메타/상단 한 줄과 병행 가능. |

# 1. 모두투어 (modetour) — 상세 추출 대상

**전용 등록 API:** `POST /api/travel/parse-and-register-modetour`  
**근거 예시(픽스처):** `가는편: 인천 → 연길 2026.04.20(월) 19:20 / CZ6074` (M1) · `쇼핑품목 인삼 / 쇼핑장소 백화점 / 예상소요시간 40분 / 환불여부 조건부` (M1) · `1일차/05-01/다낭/예정/A호텔` (M2)

### 본문에서 뽑아야 하는 것 (서술형)

| 대상 | 필수/선택 | 비고 |
|------|-----------|------|
| 일차별 일정 설명 | 필수(있으면) | `schedule[]` |
| 관광지·식사·현지교통 | 선택 | |
| 포함/불포함 | 필수(있으면) | `included` / `excluded` |
| 주의·미팅·공항 안내 | 선택 | `mustKnow` 등 |
| 쇼핑센터 방문 서술 | 선택 | 쇼핑 **행**과 혼동 시 정형 쇼핑칸 우선 |
| 선택경비·안내만 | 선택 | 옵션 **데이터 행**과 구분 (M2류) |

**넣으면 안 되는 것:** 정형칸이 채운 항공·호텔·옵션·쇼핑 **최종 필드 덮어쓰기**. 쇼핑 환불 칸에 법률·권리 **장문** 넣기(M3 회귀 기준 오염).

### 항공

- **형태:** 항공사 1줄 + `가는편`/`오는편` 또는 `출발`/`입국` 라인; Open-jaw(A→B, 귀국 C→A) 허용.
- **슬롯:** 항공사, 출발/도착 도시·일시, 편명, 공항(선택), 소요시간(선택). 날짜 `YYYY.MM.DD(요일)`·`MM-DD` 혼재 허용.
- **충돌:** 정형칸 > 본문 구조화.

### 호텔

- 일차·날짜·도시·호텔명 **반복 행**; 동일 호텔 연박 반복 유지. `예정`/동급 문구는 비고.

### 옵션관광

- 옵션명, 통화, 성인/아동가, 소요시간, 최소인원, 동행, 미참여 대기. **최대 30행** 상한. 싱글차지·1인 객실 등 **비옵션**은 옵션 행에서 제외.
- **근거 예시:** `백두산 투어 USD 180 성인 소요시간 8시간` (M1)

### 쇼핑

- 행: 쇼핑항목, 대표장소, 추가장소, 소요시간, 환불여부. `shoppingVisitCount`는 요약; **리스트 우선**.

# 2. 참좋은여행 (verygoodtour) — 상세 추출 대상

**전용 등록 API:** `POST /api/travel/parse-and-register-verygoodtour`  
**근거 예시:** `출국` / `2026.07.15 (수) 12:35 인천 출발` (C4) · `번호 / 선택관광명 / 내용 / 비용 / 시간 / …` 표 (C4) · `숙박시설 미정이며 출발 전까지 홈페이지를 통해 안내드립니다` (C4)

### 본문

여행요약, 일차별 이동·관광·식사, 포함/불포함, 유의, **호텔 설명형·미정 정책**, TGV/열차 등.

### 항공

**출국 / 입국** 블록 + 시각 줄(2줄형) 또는 `가는편`/`오는편` 한 줄형.

### 호텔

미정·등급·인원 규정 **설명형**; 일자+확정 호텔 행이 있으면 표와 병행. (verygood 계약상 미정-only는 행 0건 정책과 연동.)

### 옵션

**번호 표**: 번호, 옵션명, 내용, 비용, 시간, 미참가 대기·장소, 동행.

### 쇼핑

**총횟수 + 회차 리스트** 또는 슬래시 요약 한 줄; 안내만 장문이면 **행 0** (C6 계약).

# 3. 하나투어 (hanatour) — 상세 추출 대상

**전용 등록 API:** `POST /api/travel/parse-and-register-hanatour` (내부적으로 레거시 other-brands·generic 항공 힌트 사용 — **축소 대상**, 아래 §7)  
**근거 예시:** `출발: 인천 → 방콕 2026.09.10 12:10 / TG659` (H1) · `도시 방콕 / 쇼핑샵명(위치) ICON / 품목 잡화 / …` (H1) · `예정호텔은 현지 사정에 따라 변경될 수 있습니다.` (H2 — 호텔명 행 아님)

### 본문

일정·관광·식사, 포함/불포함, 주의, 가이드/미팅, 호텔 소개 장문, 항공·입국 안내.

### 항공

**출발편/귀국편** 요약 한 줄 + 상세 다줄 **공존**. 충돌 시 시간·편명·공항이 더 채워진 쪽 유리.

### 호텔

**예정호텔 다중 후보**: 후보 목록, 객실기준, 확정시점, 설명.

### 옵션

**블록 반복**: 옵션명, 성인/아동가, 소요시간, 대체일정, 미선택 시 가이드동행, 비고.

### 쇼핑

도시, 쇼핑샵명, 위치, 품목, 소요시간; 횟수는 별도 필드.

# 4. 노랑풍선 (ybtour) — 상세 추출 대상

**전용 등록 API:** `POST /api/travel/parse-and-register-ybtour` (레거시 URL `…-yellowballoon`은 동일 핸들러로 리다이렉트성 유지 — **축소 대상**, §7)  
**근거 예시:** `가는편: 김포 → 오사카 2026-07-01 08:30 / KE721` (Y1) · `1. 유니버설 스튜디오 / 비용 120USD / 시간 6시간 / 미참가시 대기일정 자유시간` (Y1) · `회차 1 | 쇼핑 품목 잡화 | 쇼핑 장소 면세점 | …` (Y1)

### 본문

일정·관광, 포함/불포함, 주의, 상품 소개. 옵션 vs 쇼핑은 **정형칸으로 축 분리**(본문에서 형식이 비슷해도).

### 항공

**출발편 / 귀국편** 블록; 편명·도시·일시.

### 호텔

DAY/구간 행 또는 설명만; 미정·비고 허용.

### 옵션

비용, 소요시간, 미참여, 동행, 비고.

### 쇼핑

**총횟수 + 회차** 리스트; 환불 장문은 본문·유의로, 행 환불 칸 오염 방지(Y3).

## 7. 공용·레거시 잔재 — 분류 (보류 / 축소 대상 / 삭제 후보)

**분류 정의**

| 분류 | 의미 |
|------|------|
| **보류** | 당장 제거·대체 불가. 참조가 넓음. **전용 파이프가 제품 규약의 SSOT**이며, 본 항목은 잔여 의존성. |
| **축소 대상** | 전용 모듈로 흡수·중복 제거·슬림화 검토. 파일 단위 삭제가 아니라 **정리 우선**. |
| **삭제 후보** | 잔여 호출부 이관·검증 후 **파일 제거**를 목표로 함. |

**실행 코드 (공용·레거시)**

| 경로 | 분류 | 설명 |
|------|------|------|
| `app/api/travel/parse-and-register/route.ts` | **삭제 후보** | 교원·기타·내부 스크립트 등 **fallback** 전용. 이들을 전용/별도 진입으로 이관한 뒤 제거 검토. |
| `lib/parse-and-register-handler.ts` | **축소 대상** | 하나·노랑 **전용 route**와 공용 route가 공유. 모두·참좋은은 **별도 전용 핸들러** — 여기서 **공용을 계속 쓴다는 의미는 없음**. |
| `lib/register-parse.ts` | **보류** | 모든 `parseForRegister*` 래퍼의 LLM·병합 본체. 무단 삭제 시 등록 전반이 중단. |
| `lib/detail-body-parser-legacy-register.ts` | **보류** | `presetDetailBody` 없는 경로·회귀. 모두·참좋은 전용 parser와 병존. |
| `lib/detail-body-parser-legacy-other-brands.ts` | **축소 대상** | 하나·노랑이 위임 중. **브랜드 전용 detail-body**로 이전 시 축소. |
| `lib/flight-preferred-legs-legacy-other.ts` | **축소 대상** | legacy-other-brands 전용 힌트. 상동 이전 시 함께 축소. |

**문서·도구 (참고)**

| 대상 | 분류 | 설명 |
|------|------|------|
| `docs/admin-register-supplier-precise-spec.md` · `docs/ADMIN-REGISTER-SINGLE-UX.md` 등 | **축소 대상** | 과거 단일 엔드포인트 서술 제거·전용+fallback 정합은 **문서 정리**로 처리. |
| `scripts/verify-detail-body-parser-regression.ts` | **보류** | 회귀용; 삭제 대상 아님. |

**명시:** 어댑터·스크래퍼·Prisma 스키마·공개 상세 소비와 무관한 임의 모듈은 본 “잔재” 목록에 넣지 않는다.

| 2 | **출발일 / 귀국일** | 첫 출발·마지막 귀국이 달력·히어로·duration resolver와 어긋나지 않는지 (M1, M2, M3). |
| 3 | **`flight_info`** | M3처럼 항공사+출발/도착 시각만 있는 경우 **과도한 required**·`sectionReview` 불일치 없는지. |
| 4 | **타사 문자열 오염** | `brandKey === modetour` 경로에서 제목·항공·옵션에 **타 브랜드 보일러플레이트**가 섞이면 교정·이슈로 분리. |
| 5 | **가격 4슬롯** | 성인·아동·엑베·노베드·유아 매핑; **유아=성인 오탐** 방지. 옵션 인접 가격 한 줄(M1)과 표 가격 정합. |
| 6 | **옵션 30개** | `OPTIONAL_TOUR_UI_MAX_ROWS`·미리보기 문구·저장 캡; **도시별 반복·중복 제거 금지** 정책과 충돌 없음. |
| 7 | **쇼핑 횟수 / 리스트** | `쇼핑횟수 총 N회`(M3)만 있고 행이 없거나, 장문만 있는 경우 **리스트 깨짐 없음**; 횟수는 요약 정합. |
