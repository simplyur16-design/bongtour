# 관리자 상품 등록 실검증 체크리스트

등록 화면(`/admin/register`)에서 **모두투어** 또는 **참좋은여행** 선택 후 아래 순서로 확인합니다.  
미리보기/confirm 응답에 `registerVerification`이 포함됩니다.

## 공통 절차

1. 본문 + 필요 시 정형칸 입력 → **AI 실시간 분석 시작**
2. **실검증 패널**(보라색 접기)에서 표·지문·fingerprint 확인
3. **최종 등록** 후 **저장 직후 실검증**에서 fingerprint 일치 여부 확인
4. 링크 **관리자 상품 화면 (`?registerTrace=1`)** → 편집 모드에서 공개 소비 안내 + 기존 구조화 블록 확인
5. **공개 상세** 새 탭에서 사용자 화면과 비교

## 모두투어

- [ ] 항공: 실검증 패널에 항공사·편명·가는/오는 줄·공항·일시가 채워지는가
- [ ] 가격: `productPriceTable` 성인·아동 엑베·노베·유아(4슬롯)가 숫자로 보이는가
- [ ] 호텔: 표에 일차·날짜·도시·호텔명·예정/확정이 dayLabel만 아닌가
- [ ] 옵션: 행 수·옵션명·가격·시간·동행/대기가 보이는가
- [ ] 쇼핑: 방문 횟수·행 수·품목·장소·환불 문구가 보이는가
- [ ] 검수: 필드 점검에서 **교정** 클릭 시 서랍에 근거 스니펫 + **실검증 추적 힌트**가 보이는가
- [ ] confirm: `structuredFingerprint`가 미리보기와 같은지(다르면 교정/Pexels 등으로 parsed 변경됨)
- [ ] 저장: `storedStructuredSignalsPreview`에 항공·가격 요약이 있는가
- [ ] 공개: `?registerTrace=1` 안내와 공개 상세 실제 화면이 어긋나지 않는가

## 참좋은여행 (verygoodtour)

- [ ] 가격: 실검증의 3슬롯(성인·아동·유아 정책)과 `verygoodPriceSsotNote`(draft) 설명이 맞는가
- [ ] 항공·호텔·옵션·쇼핑: 위 모두투어 항목과 동일하게 패널에서 눈으로 확인
- [ ] fingerprint·저장 요약·공개 추적: 모두투어와 동일 절차

## 노랑풍선 / 하나투어 / 공용 API

- 현재 `registerVerification`은 **전용 API**(모두·참좋은) 응답에만 포함됩니다. 해당 공급사는 기존 미리보기 UI·필드 점검·교정 서랍만 사용합니다.

## 알려진 블라인드

- 공개 상세는 브랜드별 Flight 노출 정책·resolver에 따라 패널의 텍스트와 카드 UI가 다를 수 있음 → `?registerTrace=1` + 구조화 가시화 블록으로 소스 추적
- 이미지 전용 가격표(텍스트 없음)는 패널에 숫자가 비어 있을 수 있음
