# 보안 이상징후 운영 런북

## 대상 이벤트
- `401` 연속: 인증 없는 관리자 API 접근 시도
- `403` 연속: 권한 없는 접근 시도
- `429` 급증: rate limit 초과 요청 급증
- `expensive` 급증: 고비용 관리자 API 반복 호출

## 알림 입력
- 소스: `admin-api-security-anomaly` 구조화 로그 + webhook payload
- 핵심 필드: `ip`, `path`, `unauthorized`, `forbidden`, `limited`, `expensive`

## 대응 역할
- 1차 확인: 운영 담당자(당번)
- 2차 기술 판단: 백엔드 담당
- 3차 차단/해제 승인: 관리자 권한자

## 대응 순서
1. 알림 수신 후 5분 내 로그 확인 (`ip`, `path`, 호출량 추세)
2. 정상 트래픽 여부 판단
   - 배포 직후/운영 작업 여부
   - 특정 UI 액션과 연동된 정상 burst인지 확인
3. 비정상으로 판단되면 임시 조치
   - WAF/IP 차단 또는 rate limit 강화
   - 고비용 API 임시 차단(필요 시)
4. 영향 범위 점검
   - 관리자 화면 지연/실패 여부
   - 인증 우회 시도 흔적 여부
5. 사후 조치
   - 임계치 조정
   - 정책/코드 보완 PR 생성
   - 사건 기록(발생시각, 원인, 조치, 재발방지)

## 판별 기준 (초안)
- `401`만 높고 `403/429` 낮음: 봇 스캔 가능성
- `403` 급증: 권한 오용 또는 계정 설정 문제 가능성
- `429` 급증: 과도한 재시도/자동화 스크립트 가능성
- `expensive` 급증: 비용성 API 오남용 가능성

## 즉시 조치 템플릿
- 제목: `[BONGTOUR] Admin API anomaly 대응 시작`
- 포함:
  - 감지 시각
  - 주요 path/ip
  - 1차 판단
  - 임시 조치
  - 다음 업데이트 ETA
