# 노랑풍선 F12 실전 검토 체크리스트

Chrome / Edge 개발자 도구로 **출발일 팝업·네트워크·일정 탭** 실측할 때 사용한다.  
식별축 판별은 [YELLOW-BALLOON-ID-CHECKLIST.md](./YELLOW-BALLOON-ID-CHECKLIST.md)와 함께 본다.

---

## 사전 준비

- [ ] Chrome 또는 Edge에서 **개발자 도구** 연다 (F12)
- [ ] **Network** 탭 연다
- [ ] **Preserve log** 켠다
- [ ] **Disable cache** 켠다
- [ ] **Fetch / XHR** 필터로 우선 본다
- [ ] 필요 시 **Doc** / **JS** / **Img** 도 보조 확인

---

## A. 상세 페이지 진입 직후

1. [ ] **현재 URL** 저장 (전체 복사)
2. [ ] **도메인** 확인
   - `ybtour.co.kr` 인가?
   - `pkg.ybtour.co.kr` 인가?
3. [ ] **HTML 소스** 또는 **script**에 상품 **대표 코드 후보**가 있는지 확인
4. [ ] 초기 로딩 **XHR** 중 상품 상세 데이터처럼 보이는 요청이 있는지 확인
5. [ ] 응답에 아래가 있는지 본다
   - `title` / 상품번호 / 일정 / 가격 / 출발일 관련 값

---

## B. 출발일 변경 버튼 클릭

1. [ ] 클릭 직전 **Network 로그 클리어**
2. [ ] **Fetch / XHR** 만 보이게 하고 **출발일 변경** 클릭
3. [ ] 새로 생긴 요청 확인
4. [ ] 아래 요청은 **우선 제외**
   - googletagmanager
   - analytics
   - beacon
   - 광고·추적 픽셀
5. [ ] **자체 도메인** 요청을 우선 확인
6. [ ] 응답 **타입** 판별
   - JSON
   - HTML fragment
   - 전체 문서
7. [ ] 응답에 아래가 있는지 확인
   - 날짜 목록
   - 가격
   - 상태
   - 좌석
   - **행 식별값**
   - 월 정보

---

## C. 출발일 팝업이 열린 뒤

### C-1. 좌측 달력 DOM

- [ ] 각 날짜 셀에 `data-date` 같은 값이 있는가?
- [ ] **가격**이 셀에 직접 들어 있는가?

### C-2. 우측 리스트 DOM

- [ ] 각 row에 `data-*` 속성이 있는가?
- [ ] `onclick` / hidden input / **링크 파라미터**가 있는가?
- [ ] **row별 식별값**이 따로 있는가?

### C-3. 스크롤·로딩

- [ ] 스크롤 시 **추가 요청**이 나는가?
- [ ] **가상 스크롤** 여부 확인
- [ ] DOM에 **일부 row만** 있는지 확인

### C-4. row 클릭

- [ ] row **하나 클릭** 시 추가 요청이 나는가?
- [ ] **상세 출발정보 조회형**인지 확인

---

## D. 이전달 / 다음달 이동

1. [ ] 월 이동 버튼 클릭 **전** 로그 클리어
2. [ ] **이전달** / **다음달** 각각 눌러 **새 요청** 확인
3. [ ] 요청 파라미터에 `year` / `month` 혹은 **date range** 가 있는지 확인
4. [ ] 응답이 **달력만** 바꾸는지, **우측 리스트도 같이** 내려주는지 확인
5. [ ] 같은 row가 **중복**으로 나타나는지 확인
6. [ ] 월 이동 시 **originCode 후보**가 바뀌는지, **유지**되는지 확인

---

## E. 탭 구조 확인

1. [ ] **상품정보·일정** 탭 클릭
2. [ ] **포함·불포함·약관** 탭 클릭
3. [ ] **옵션·쇼핑·관광** 탭 클릭
4. [ ] **리뷰** 탭 클릭
5. [ ] 각 탭마다 아래 확인
   - 정적 HTML 인지
   - 클릭 시 **XHR / fetch** 가 나는지
   - **raw block** 을 그대로 저장할 수 있는지
   - **일정 탭에만** day 단위 분해 가능한 **정본**이 있는지

---

## F. 일정 탭 (가장 중요)

1. [ ] 일정 탭 클릭 후 **raw block** 확보 가능 여부 확인
2. [ ] **1일차 / 2일차 / 3일차** 구분 DOM 이 있는지 확인
3. [ ] 도시 흐름 / 식사 / 숙소 / 방문지 텍스트가 **분리 가능**한지 확인
4. [ ] 일정 탭 **실패** 시 `ItineraryDay` 는 **비워야** 하며, **다른 요약 문구로 대체 생성하지 않는다**

---

## G. 금지 해석

| 하지 말 것 | 이유 |
|------------|------|
| GTM iframe 반응 = 출발일 **데이터 소스** 라고 판단 | 추적 스크립트일 뿐 |
| 달력 셀 가격 = **최종** `ProductDeparture` 가격 이라고 단정 | 우측 리스트·API가 정본일 수 있음 ([설계안](./YELLOW-BALLOON-ADAPTER-DESIGN.md) 참고) |
| 핵심포인트·리뷰·요약 = `ItineraryDay` **대체 정본** | 일정 탭·일정 원문만 정본 |
| 화면 상품번호 = `originCode` **즉시 확정** | [식별 체크리스트](./YELLOW-BALLOON-ID-CHECKLIST.md) 필수 |
