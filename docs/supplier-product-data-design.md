# 공급사 여행상품 상세페이지 데이터 수집 설계

봉투어는 공급사(하나투어, 모두투어 등) 상품을 **검수·정리·상담 연결**하는 창구다. 공급사 원문 상품 정보와 일정표는 **수정/재구성하지 않는다**. 이 문서는 공급사 상세페이지에서 **무엇을 안정적으로 읽어올 수 있는지**를 정리하고, 봉투어 내부 **표준 수집 리스트**와 **우선순위**를 정의한다.  
(본격 스크래퍼 구현 전 **정보 구조 분석 + 수집 우선순위 정의**가 목표다.)

---

## 1) 이 작업이 필요한 이유

- **등록·목록·상담**에 필요한 최소 정보(상품코드, URL, 상품명, 기간, 목적지, 가격·일정 등)를 **어디까지 자동 수집**할지 기준이 없으면, 여행사별로 추출 항목이 제각각이 되고 유지보수 비용이 커진다.
- **URL/페이지 구조**가 여행사마다 다르므로, “추출 가능 / 탭·AJAX 필요 / 달력·API 필요”를 미리 구분해 두어야 **구현 난이도**와 **안정성**을 예측할 수 있다.
- **필수 1차 / 권장 2차 / 고급 자동화 3차**로 우선순위를 나눠 두면, MVP는 1차만 수집하고 2·3차는 단계적으로 확장할 수 있다.
- 조사표 템플릿을 두면 **여행사별 차이**를 체계적으로 기록하고, 신규 공급사 추가 시 동일한 관점으로 조사할 수 있다.

---

## 2) 공급사 페이지에서 추출 대상이 되는 정보 — 큰 분류

| 분류 | 설명 | 봉투어 내부 대응 필드 예시 |
|------|------|----------------------------|
| **식별·URL** | 상품 고유 식별, 원문 링크 | `originCode`, `originUrl`, `originSource`, `brandId` |
| **기본 정보** | 상품명, 기간, 목적지, 항공 등 | `title`, `duration`, `destination`, `airline` |
| **가격·요금** | 성인/아동/유아, 유류·제세·가이드비, 현지 필수 | `ProductPrice`, `mandatoryLocalFee`, `mandatoryCurrency`, `isFuelIncluded`, `isGuideFeeIncluded` |
| **일정·포함/불포함** | 일차별 일정, 포함/불포함 텍스트, 상담용 요약 | `schedule`, `includedText`, `excludedText`, `criticalExclusions` |
| **쇼핑·선택관광** | 쇼핑 횟수/품목, 선택관광 목록 | `shoppingCount`, `shoppingItems`, `OptionalTour` |
| **상태·예약** | 출발확정, 잔여좌석, 예약 가능 여부, 프로모션 | (현재 스키마에는 일부만; 3차 수집 후 확장) |
| **부가 메타** | 호텔, 리뷰/평점, 해시태그, 이미지 유무 | 2·3차 수집·분류용 |

## 3) 봉투어 표준 수집 리스트

### 필수 1차 수집 (등록/목록/상담에 반드시 필요)

| 항목 | 설명 | 봉투어 필드 | 비고 |
|------|------|-------------|------|
| 상품 URL | 공급사 상세 페이지 URL | `originUrl` | 중복·추적·재수집 기준 |
| 상품코드 | 공급사 부여 상품 고유 코드 | `originCode` | 동기화 키, @unique |
| 상품명 | 공급사 표기 상품명 | `title` | 원문 유지 |
| 여행사 표시명 | 하나투어, 모두투어 등 | `originSource` / Brand | 목록·필터용 |
| 기간 | N박N일 등 | `duration` | 원문 유지 |
| 목적지 | 국가/도시 | `destination` | 원문 유지 |
| 항공사 | 있을 경우 | `airline` | 원문 유지 |
| 포함/불포함 | 포함·불포함 내역 전체 텍스트 | `includedText`, `excludedText` | 원문 유지 |
| 상담용 불포함 요약 | 가이드비·유류 등 핵심만 | `criticalExclusions` | 상담용 |
| 일정표 | 일차별 일정(텍스트/구조) | `schedule` (JSON) 또는 Itinerary | 원문 유지, 표시 SSOT |
| 현지 필수 지불액 | 가이드비 등 USD/EUR 등 | `mandatoryLocalFee`, `mandatoryCurrency` | 상담·결제 안내 |
| 유류/가이드비 포함 여부 | 포함 여부 플래그 | `isFuelIncluded`, `isGuideFeeIncluded` | 있을 경우 |

### 권장 2차 수집 (품질·노출·검수 향상)

| 항목 | 설명 | 봉투어 필드 | 비고 |
|------|------|-------------|------|
| 쇼핑 횟수/품목 | 쇼핑 방문 횟수, 품목 요약 | `shoppingCount`, `shoppingItems` | 노쇼핑 표시 등 |
| 선택관광 | 이름, 가격(USD), 소요시간, 미참여 시 대기장소 | `OptionalTour` | 상담·선택 시 필요 |
| 첫 출발일 가격 | 성인/아동/유아 등 | `ProductPrice` 1 row | 목록·요약용 (달력 없이) |
| 단체번호/운영코드 | 공급사 내부 코드(있으면) | (추가 시) | 상담·연계용 |
| 출발확정 여부 | 있으면 | (추가 시) | 노출·필터용 |
| 특전/핵심 포인트 | 프로모션 문구 등 | `counselingNotes` 등 | 검수·노출 보조 |

### 고급 자동화 3차 수집 (가격·달력·잔여·상태 추적)

| 항목 | 설명 | 봉투어 필드 | 비고 |
|------|------|-------------|------|
| 출발일별 가격 | 달력/탭별 성인·아동·유아·현지금 | `ProductPrice` (날짜별) | 달력 UI, 가격 변동 추적 |
| 유류할증료/제세공과금 | 일별 또는 상품별 | (확장) | 가격 투명성 |
| 출발/도착 시간·공항·편명 | 있을 경우 | (확장) | 상세 노출 |
| 최소 출발 인원 / 잔여 좌석 | 있으면 | (확장) | 예약 가능성 표시 |
| 예약 가능 여부 | API/버튼 상태 등 | (확장) | 실시간성 |
| 호텔 정보 | 호텔명·등급·지역 등 | (확장) | 검색/필터 |
| 리뷰 수/평점 | 있으면 | (확장) | 노출·정렬 |
| 프로모션/이벤트 여부 | 배너·라벨 등 | (확장) | 노출 |

## 4) 여행사별 조사표 템플릿

아래 표를 **여행사별로 1장씩** 채워 나간다.

| 항목 | 내용 |
|------|------|
| **여행사명** | (예: 하나투어, 모두투어) |
| **상품 상세 URL 패턴** | (예: `https://www.hanatour.com/.../package/detail?pkgCd=ATP...`) |
| **상품코드 위치** | URL 쿼리 / path / 메타 / 본문 특정 요소 |
| **날짜/가격 구조** | 단일 상품형 / 날짜 선택형, 달력·탭 여부, 별도 API 호출 여부 |
| **일정표 구조** | 한 페이지 내 블록 / 탭 전환 / 아코디언 / 테이블 vs 리스트 |
| **항공 정보** | 본문 고정 / 탭 / 별도 iframe·API |
| **가격 추출 난이도** | 1=HTML만 / 2=탭·클릭 / 3=달력·AJAX / 4=공급사별 커스텀 |
| **좌석/예약 상태** | 추출 가능 여부, 필요 시 API·쿠키 등 |
| **비고** | 로그인 필요, 캡차, rate limit, CORS 등 |

### 조사표 예시 (빈 템플릿)

```markdown
## [여행사명]

| 항목 | 값 |
|------|-----|
| URL 패턴 | |
| 상품코드 위치 | |
| 날짜/가격 구조 | |
| 일정표 구조 | |
| 항공 정보 위치 | |
| 가격 추출 난이도 | 1 / 2 / 3 / 4 |
| 좌석/예약 상태 | |
| 비고 | |
```

## 5) 항목별 난이도/안정성 분류 기준

| 단계 | 정의 | 대응 전략 |
|------|------|-----------|
| **1단계** | **HTML에서 안정적으로 추출 가능** — 초기 로드된 정적 HTML에 텍스트/속성으로 존재, selector만으로 읽을 수 있음 | Playwright 등으로 HTML 로드 후 파싱만으로 처리, 유지보수 낮음 |
| **2단계** | **탭 클릭/추가 렌더링 필요** — 일정·가격 등이 탭/아코디언 안에 있어 클릭·대기 후 DOM에 나타남 | 클릭 시퀀스 + 로딩 대기 후 추출, 여행사별 selector 관리 |
| **3단계** | **달력/AJAX/API/비동기 응답 필요** — 날짜 선택 시 XHR/fetch로 가격·잔여 등 로드 | 네트워크 후킹 또는 API 역분석, 불안정·변경 리스크 있음 |
| **4단계** | **공급사별 커스텀 처리 필요** — 캡차, 로그인, 특수 인코딩, 앱 전용 API 등 | 공급사별 전용 플로우, 문서화·모니터링 필요 |

- **안정적**: 1단계 또는 2단계로 일관되게 동작하는 항목  
- **조건부 가능**: 2·3단계로 특정 조건(탭/날짜 선택 등)에서만 추출 가능  
- **난이도 높음**: 3·4단계, 공급사 UI/API 변경 시 깨지기 쉬움  

---

## 6) 먼저 조사해야 할 공급사 우선순위

1. **하나투어** — 봉투어에서 이미 `originSource`/Brand로 사용 중인 것으로 보임, 상품코드(ATP/AVP 등)·`originUrl` 구조 파악 우선.  
2. **모두투어** — 동일하게 주요 공급사로 가정 시, URL 패턴·일정표·가격 구조 조사.  
3. **기타 브랜드** — `Brand` 테이블에 추가 예정인 여행사 순으로, 동일 조사표로 조사.

우선 **1~2개 공급사**에 대해 위 조사표를 채우고, **필수 1차 수집 항목**이 각 단계(1~4) 중 어디에 해당하는지 매핑한 뒤, 스크래퍼 구현 범위를 정하는 것을 권장한다.
