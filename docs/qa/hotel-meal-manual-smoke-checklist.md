# 호텔·식사 수동 스모크 (관리자 기준)

**순서**

- [ ] 1. 공급사 상세 **pasted raw** 준비 (필수 본문 + 필요 시 섹션 붙여넣기)
- [ ] 2. **Admin → 등록**에서 미리보기 생성 후 **preview**에서 호텔/식사 블록·일차 행 확인
- [ ] 3. **confirm 저장** (최종 저장)
- [ ] 4. **상세페이지** (`/products/[id]`) 일정 요약에서 호텔/식사 카드 확인
- [ ] 5. **DB** `Product`·`ItineraryDay` 해당 컬럼 값 확인 (관리자 도구·DB 클라이언트)
- [ ] 6. **기존 상품** 1건: 필드 대부분 null이어도 상세 **오류 없음**·빈 카드만 난잡하지 않음

**케이스별 (샘플 또는 단계별로 반복)**

- [ ] 호텔명 **2개 이상** → 「대표명 외 n」
- [ ] 호텔명 **1개** → 단일명, 「외 0」 없음
- [ ] 식사 **아침·점심·저녁** 모두 → 가로 한 줄
- [ ] 식사 **일부만** → 채워진 끼만 표시
- [ ] **mealSummaryText**만 (분리 실패) → fallback 한 덩어리, 끼별과 중복 없음
- [ ] **day.hotelText 없음** → 상품 요약/이름만으로 day 줄이 채워지는지(의도대로)·상단 호텔 블록과 역할 구분
- [ ] **기존 구데이터** (null 다수) → 크래시 없음

**참고:** 코드 상수 `lib/register-flow/hotel-meal-e2e-checklist.ts`
