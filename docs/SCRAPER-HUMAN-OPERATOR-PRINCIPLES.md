# 스크래퍼 공통 원칙 — 사람처럼 읽고 넘기는 수집기

이 프로젝트의 스크래퍼는 **기계적으로 빠르게 긁는 봇**이 아니라, **실제 사용자가 화면을 보며 읽고 클릭하는 흐름**에 가깝게 동작해야 한다.

**범위**: 위 원칙은 **모든 공급사·모든 사이트 대상 스크래퍼**에 적용한다 (하나·모두·노랑·참좋은·달력 E2E·기타 전용 스크립트 포함).

## 공급사별 설정 — 통합 금지

- **사람처럼 동작**하는 것은 전 공급사 **공통 목표**이지만, **지연·간격·재시도·병렬도·단계 정의**는 사이트마다 다르다.
- **절대 통합하지 않는다**: “전 공급사 공통 `DELAY_MS` / 공통 `scraper_config.yaml` 하나”처럼 **한 설정으로 여러 공급사를 묶지 않는다.**
- **각 공급사마다 따로 둔다**: 해당 공급사 전용 디렉터리·모듈·상수·환경 변수만으로 그 사이트의 간격과 정책을 정의한다. 다른 공급사와 값을 공유하려면 복붙이 아니라, 필요 시에만 **의도적으로** 각 쪽 파일에 숫자를 적되, **런타임에 하나의 객체를 공유하는 SSOT**로 만들지 않는다.
- **공통 헬퍼**(`scripts/scraper_common/human_operator.py` 등)는 **구현 재사용**용이지, **전사 단일 타이밍 설정**이 아니다.

## 최상위 목표

1. 차단당하지 않도록 **자연스럽게** 동작할 것  
2. **사람의 실제 사용 흐름**을 최대한 닮을 것  
3. **과속 요청·과도한 반복·비정상 패턴**을 피할 것  
4. 사이트 구조를 억지로 밀어붙이지 말고, **화면 상태**를 보고 다음 행동을 결정할 것  

**속도보다 안정성**, **대량 처리보다 자연스러운 동작**, **강제 추출보다 사람처럼 읽는 흐름**을 우선한다.

## 공통 동작 원칙

1. **페이지 진입 후** 즉시 DOM 전체를 과도하게 긁지 말 것. 렌더될 시간을 줄 것. 상단 → 본문 → 탭 → 리스트 순으로 접근.  
2. **클릭 전** 요소 visible·enabled·로딩·모달·이전 액션 완료를 확인.  
3. **고정 sleep 남발 금지** — 대기는 DOM/텍스트/row/탭/로딩/모달 상태 변화 우선, 필요 시만 짧은 **랜덤 간격** 안정화.  
4. **스크롤/드래그**는 한 번에 끝까지 순간이동하지 말고, 여러 번 나누고 중간 상태를 확인.  
5. **동일 액션을 짧은 시간에 과도 반복하지 말 것.** 재시도는 즉시 연속 난사하지 말고, 원인 확인 후 **제한적** 재시도.  
6. 가능하면 **네트워크보다 화면·DOM 상태**를 기준으로 판단.

## 절대 금지 (요약)

초고속 연속 클릭/스크롤, 진입 직후 대량 selector 난사, 동일 실패 무한 반복, sleep만 믿고 진행, 화면 안 바뀌는데 강행, 첫 화면 row만 보고 끝내기, 기계적 고정 간격, 차단 징후 무시 강행.

## 대기 우선순위

1. DOM 상태 변화  
2. 텍스트 변화  
3. row 수 변화  
4. active tab/class  
5. loading/skeleton 사라짐  
6. modal open/close  
7. 필요 시 **짧은 추가 안정화** (랜덤)

## 실패 처리

실패를 숨기지 말고 단계를 로그로 남길 것. 무한 재시도 금지. 403/캡차/접속 제한 조짐이면 **속도 낮추기·중단·보수적 전환**. fallback이 있어도 **먼저 실패 원인**을 남길 것.

## 공통 로그 이벤트

`[SCRAPER_EVENT] ...` — `scripts/scraper_common/human_operator.py` 의 `EV_*` 및 `scraper_log()` 참고.

## 구현 참고

- **공통 헬퍼**: `scripts/scraper_common/human_operator.py` (`human_scroll_read_pause`, `human_stabilize_short`, `wait_for_detail_key_area`, `scraper_log`, 차단 문구 감지 등) — **패턴·유틸만** 공유, 간격 기본값을 전 공급사에 강제하지 않는다.  
- **기존 인간 지연(달력 E2E 전용)**: `scripts/calendar_e2e_scraper/utils.py` 의 `human_delay`, `product_interval`, `pagedown_delay` — **그 스크립트·공급사 맥락 전용**으로 두고, 다른 공급사 스크래퍼의 “단일 설정 소스”로 끌어다 쓰지 않는다.

새 공급사 스크래퍼는 위 **행동 원칙**을 따르고, **타이밍·설정은 그 공급사 전용 파일**에 둔다.
